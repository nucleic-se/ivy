# Library

Permanent knowledge base. Findings, patterns, references, and insights worth keeping beyond any individual project.

**The library is permanent. Projects are transient.** Before archiving a project, migrate anything with long-term value here. Knowledge left only in a project directory is knowledge that will be lost.

## Contents
- `index.md`: This file.

## Organisation

Add domain subdirectories as content accumulates. Each subdirectory needs an `index.md`. Suggested starting structure:
```
library/
├── index.md
├── <domain>/
│   ├── index.md
│   └── brief-NNN-title.md
```

## Curation Standards

- Each library entry should be self-contained: title, source, key findings, applicability.
- Prefer breadth briefs (key ideas in 200–500 words) over verbatim copies.
- Tag entries with domain keywords in the `index.md` entry for searchability.
- Remove entries that are superseded or no longer relevant — the library is curated, not archived.
